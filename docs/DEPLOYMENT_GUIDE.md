# ChildGuard Deployment Guide

This guide covers the recommended, safe and fast way to deploy ChildGuard on Windows.

## 1) Portable Package (Recommended for trial/POC)

- File: `dist/ChildGuard_Portable_<version>.zip` (generated by `scripts/build_portable.ps1`)
- Contents:
  - `out/UI/ChildGuard.UI.exe` — main UI (self-contained, single-file)
  - `out/Agent/ChildGuard.Agent.exe` — background agent (self-contained, single-file)
  - `tools/` — helper scripts for autostart (optional)

Steps:
1. Extract the ZIP to a writable folder (e.g. `C:\Apps\ChildGuard`)
2. Double-click `out/UI/ChildGuard.UI.exe` to configure settings, language, theme
3. Run `out/Agent/ChildGuard.Agent.exe` for background monitoring
4. To auto-start Agent at logon, choose one of:
   - Current user (no elevation typically required):
     - `powershell -ExecutionPolicy Bypass -File installer\tools\ensure_agent_autostart.ps1 -Mode current -TaskName ChildGuardAgent -ExePath "C:\Apps\ChildGuard\out\Agent\ChildGuard.Agent.exe"`
   - All users (requires proper privileges):
     - `powershell -ExecutionPolicy Bypass -File installer\tools\ensure_agent_autostart.ps1 -Mode allusers -TaskName ChildGuardAgent -ExePath "C:\Apps\ChildGuard\out\Agent\ChildGuard.Agent.exe"`

Notes:
- The script will fallback to HKCU Run if Scheduled Task creation is blocked.
- Logs and config default to ProgramData with LocalAppData fallback.

## 2) Installer Package (Recommended for production)

Prerequisites:
- Inno Setup 6 installed, `ISCC.exe` available in PATH.

Build steps:
- `powershell -ExecutionPolicy Bypass -File scripts\build_installer.ps1 -Configuration Release -Rid win-x64 -Version <version>`
- Output installer copied to `dist/` as `ChildGuardSetup_<version>.exe`

Install steps:
- Run the generated installer. It publishes UI and Agent, configures autostart via Scheduled Task, with Run key fallback.

Uninstall autostart (dev/testing):
- `powershell -ExecutionPolicy Bypass -File installer\tools\uninstall_agent_task.ps1`
- or `powershell -ExecutionPolicy Bypass -File installer\tools\uninstall_agent_run.ps1`

## 3) Troubleshooting
- If Scheduled Task creation fails (0x41306 / Access Denied): use `-Mode current` or rely on HKCU Run fallback
- If scripts are blocked: use `-ExecutionPolicy Bypass` or run elevated PowerShell
- If `ISCC.exe` not found: install Inno Setup 6 and re-run installer build
- If UI not appearing on top: `scripts/bring_ui_front.ps1`
- Ensure write permissions for ProgramData/LocalAppData

## 4) CI/CD Notes
- Portable build is deterministic and doesn’t require external tools; ideal for CI
- Installer build requires Inno Setup 6 on the runner
- Suggested verification: `dotnet build -c Release`, run unit tests, publish portable, verify artifacts exist and sizes are within expected bounds
